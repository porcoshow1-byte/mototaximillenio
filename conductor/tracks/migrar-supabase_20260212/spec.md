# Specification: Migração Firebase → Supabase

**Track ID:** migrar-supabase_20260212
**Type:** Refactor

## Summary

Migração completa do backend de Firebase (Firestore, Auth, Realtime DB, Storage) para Supabase (PostgreSQL, Auth, Realtime Broadcast, Storage), sem alterar o comportamento funcional do app.

## Context

O MotoJá atualmente usa Firebase como BaaS com 4 serviços:
- **Firebase Auth** → login/registro de passageiros, pilotos, admins e empresas
- **Firestore** → banco de dados NoSQL para corridas, usuários, empresas, settings, tickets
- **Realtime Database** → chat em tempo real e rastreamento GPS do piloto
- **Firebase Storage** → upload de fotos (avatar, CNH, comprovantes)

A migração visa melhor suporte a dados relacionais, custos previsíveis e autonomia.

## Technical Notes

- Migração gradual: módulo por módulo, mantendo Firebase funcional até conclusão
- Padrão adapter: criar `services/supabase.ts` como ponto central (similar ao `firebase.ts`)
- RLS (Row Level Security) será configurado para cada tabela
- Chat usa Supabase Realtime; GPS usa Supabase Broadcast (sem escrita no banco)
